## Лабораторная работа №3. Работа с Markdown-разметкой и базовое использование LaTeX в документации проекта


**Markdown — это лёгкий язык разметки, который используется для оформления README-файлов, документации, отчётов и даже научных текстов. Он прост в использовании, хорошо читается как в сыром виде, так и после рендеринга, и широко поддерживается GitHub, VS Code и многими платформами.**

Благодаря своей простоте, Markdown используется повсеместно:
*   **Документация и README-файлы:** стандарт для описания проектов на GitHub и GitLab.
*   **Блоги и CMS:** многие движки (например, Ghost) и статические генераторы сайтов используют Markdown для написания постов.
*   **Заметки и личное планирование:** приложения вроде Obsidian, Typora и Joplin базируются на этом языке.
*   **Форумы и чаты:** такие платформы, как Reddit, Discord и Slack, поддерживают его для форматирования сообщений.
*   **Презентации и научные работы:** инструменты вроде Marp и Quarto позволяют создавать презентации и документы на основе Markdown и LaTeX.

Для разработчиков Markdown является незаменимым инструментом по нескольким причинам:
1.  **Скорость и простота:** позволяет писать документацию, не отвлекаясь на долгий подбор шрифтов и кнопок форматирования в интерфейсе.
2.  **Универсальность:** файлы `.md` легко конвертируются в HTML, PDF, DOCX и другие форматы для публикации или печати.
3.  **Интеграция с системами контроля версий:** идеально подходит для Git, так как изменения в тексте легко отслеживать, в отличие от бинарных форматов (например, `.docx`).
4.  **Единообразие:** гарантирует, что документация будет выглядеть одинаково хорошо на любой платформе (GitHub, VS Code, любой браузер).
